Для того чтоб включить изменения из одной ветки в другую можно использовать и git rebase, и git merge, но принцип их действия разный.
Например, если два разработчика работало над проектом, и неободимо как-то решить конфликт их коммитов:
- git merge создает коммит слияния, который наследует изменения от обоих коммитов двух разработчиков, лог коммитов как бы "раздваивается";
- git rebase просто использует один из коммитов (или серию коммитов) этих двух разработчиков - игнорируя второй, то есть лог коммитов останется "линейным" - но не учтет изменения, внесенные вторым разработчиком.
